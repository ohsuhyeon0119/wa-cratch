## 태스크

### Task 1 — 백엔드: GET /projects/:id 접근 제어 (#39) (3분)
- 파일: `backend/app/routers/projects.py`
- 구현:
  - `get_current_user` 대신 선택적 인증 `get_optional_current_user` 의존성 추가
  - `backend/app/core/security.py`에 `get_optional_current_user` 함수 추가
  - `GET /{project_id}` 엔드포인트: `published=False`인 경우 소유자가 아니면 404 반환
- 검증: `hurl --test backend/tests/hurl/projects.hurl --variable base_url=http://localhost:8000`

### Task 2 — 프론트: 로그아웃 버튼 수정 (#31) (2분)
- 파일: `frontend/src/pages/DashboardPage/DashboardPage.tsx`
- 구현:
  - `useAuth`에서 `logout` 추출
  - 로그아웃 Link의 `onClick`에서 `logout()` 호출 추가 (기존 `showToast` 유지)
  - `useNavigate`로 `/` 이동 (현재 Link to="/"가 처리하므로 동일)
- 검증: `npx playwright test e2e/dashboard.spec.ts`

### Task 3 — 프론트: ProtectedRoute + 라우트 보호 (#32) (3분)
- 파일: `frontend/src/components/ProtectedRoute/ProtectedRoute.tsx` (신규)
- 구현:
  - `localStorage.getItem('token')` 없으면 `<Navigate to="/login" replace />` 반환
  - `isLoading` 중에는 null 반환 (AuthContext의 isLoading 사용)
- 파일: `frontend/src/App.tsx`
  - `/dashboard`, `/editor/:id`, `/editor/new` 라우트를 ProtectedRoute로 감싸기
  - `/editor/new` 라우트 신규 추가
- 검증: `npx playwright test e2e/dashboard.spec.ts e2e/editor.spec.ts`

### Task 4 — 프론트: 에디터 저장 401 처리 (#33) (2분)
- 파일: `frontend/src/pages/EditorPage/EditorPage.tsx`
- 구현:
  - `handleSave` catch 블록에서 `axios.isAxiosError(err) && err.response?.status === 401` 감지
  - 401 시: `showToast('저장하려면 로그인이 필요해요 🔑', 'error')` + `navigate('/login')`
  - 그 외: 기존 `showToast('저장에 실패했어요 😢', 'error')` 유지
- 검증: `npx playwright test e2e/editor.spec.ts`
